# Prompt For GPT Or Codex: ROS Beta Promo + Docs Site

Build a compact production-ready website for ROS Beta.

Context:
- ROS is a local-first desktop operating workspace called OSA Midnight Oil / ROS Beta.
- It includes Vault Notes, Library, Research Vault, Wallet Vault, Control Room, and Nostr Lounge.
- Nostr Lounge is a sidecar social desk, not the core pitch.
- The beta is distributed as direct desktop downloads for macOS, Windows, and Linux.
- Trust, backup, recovery, and native desktop posture are central product messages.

Required pages:
1. Landing / Promo page
2. About page
3. Docs index
4. Install page
5. Trust / Local-first page
6. Beta release notes page

Design direction:
- Dark, cinematic, precise, operational.
- Avoid generic SaaS gradients and purple-heavy defaults.
- Favor screenshots, strong typography, and restrained motion.
- Reuse ROS visual language instead of inventing a separate brand.

Content goals:
- Explain ROS as a local-first desktop workspace.
- Highlight backups, recovery, native vault posture, and Control Room.
- Mention Nostr Lounge lightly as an after-hours sidecar.
- Include platform download calls to action.
- Keep docs practical: install, first launch, unlock, backup, recovery, trust, and Nostr basics.

Implementation expectations:
- Choose a simple static-friendly stack.
- Structure content so download links and screenshots are easy to update.
- Write real first-pass copy, not lorem ipsum.
- Keep mobile and desktop layouts both clean.
