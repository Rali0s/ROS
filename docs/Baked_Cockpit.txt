Active Surface States
System Surface
Shows the doctrine/module launcher surface.
Module Surface
Shows the selected module inside the cockpit.
Empty Surface
Shows a calm placeholder if no module is active.
Visual Requirements
	•	Do not show large floating app windows by default.
	•	Do not cover the cockpit with modal layers.
	•	Preserve the black-glass ROS aesthetic.
	•	Keep module content visually framed as part of the cockpit.
	•	Avoid heavy animation.
	•	Use only subtle transitions between system surface and module surface.
Compatibility
	•	Do not delete the existing windowed module system.
	•	Keep existing app components intact where possible.
	•	Wrap existing module components inside the new Active Module Surface.
	•	Add an optional “Open as Window” action for modules that still support popout behavior.
Acceptance Criteria
	•	Clicking a module opens it inside the cockpit center panel.
	•	Command Deck and Intelligence Rail remain visible.
	•	User can return to the System Surface.
	•	Existing modules still function.
	•	Optional popout/window behavior remains available.
	•	No overlapping app windows appear by default.
	•	npx eslint src passes.
	•	npm run build passes.
